# --wrap Linker Option 示例

这个示例演示了 GNU 链接器的 `--wrap` 选项的使用方法。

## 什么是 --wrap？

`--wrap` 是 GNU 链接器（ld）的一个功能，允许你在不修改原始代码的情况下拦截和替换函数调用。

## 工作原理

1. **原始函数** (`original_lib.c`): 定义了两个函数 `get_current_time()` 和 `get_random_number()`
2. **包装函数** (`wrapper.c`): 实现了 `__wrap_get_current_time()` 和 `__wrap_get_random_number()`
3. **链接器重定向**: 使用 `--wrap` 选项后，所有对原函数的调用都会被重定向到包装函数
4. **访问原始函数**: 包装函数可以通过 `__real_函数名()` 来调用原始函数

## 文件说明

- `original_lib.c`: 原始函数库（模拟外部库）
- `wrapper.c`: 包装函数实现
- `main.c`: 主程序（不需要任何修改！）
- `CMakeLists.txt`: 构建配置

## 编译和运行

```bash
cd examples/wrap_demo
mkdir build && cd build
cmake ..
make
./wrap_demo
```

## 预期输出

运行程序后，你会看到：
- 每次调用 `get_current_time()` 时，包装函数会打印日志并返回修改后的时间（+1000秒）
- 每次调用 `get_random_number()` 时，包装函数会打印日志并返回固定值 42

## 实际应用场景

1. **回放模式**: 在回放模式下替换时间函数，使用记录的时间戳而不是系统时间
2. **单元测试**: 替换难以测试的函数（如系统调用、硬件访问等）
3. **调试**: 添加日志或监控功能而不修改原始代码
4. **性能分析**: 包装函数来测量执行时间

## 注意事项

- 仅适用于 GNU 链接器（Linux），不适用于 macOS
- 需要提供 `__wrap_函数名` 的实现
- 原始函数通过 `__real_函数名` 访问
- 必须在链接阶段指定 `--wrap` 选项

## 与单元测试中的 mock 对比

| 特性 | --wrap | extern "C" mock |
|------|--------|-----------------|
| 作用范围 | 整个链接单元 | 整个可执行文件 |
